# justmd

中文 | [English](README.md)

我经常需要写markdown、读markdown然后分享markdown出去，分享对象如果是一个非程序员，常常会问我“这个怎么打开”，没想到这个问题竟然让我不知道如何回答。  

我可以说，md就是极简的标记语言，你可以用记事本打开，无视那些井号和星号，我觉得他一定会奇怪的看我，我也可以让他下载typora最后发现是收费的，又或者发过去一个在线预览链接说“你把内容粘贴到左边，右边就会显示出来”...。  

找了一圈发现一件奇怪的事，纯粹的 Markdown 客户端编辑器并不多，我可以宣扬一百遍markdown是大部分人足够可用的书写方式，并且他会大幅简化你的生活，然而证据呢？  

如果一个排版格式足够好，好到所有人都应该用一下，那么他最起码需要可以双击->打开，否则怎么能触达那剩下90%甚至99%没有使用过markdown的人类呢，这就是justmd的初心和愿景 —— Markdown 不应该需要解释如何打开。

> 一个极简的桌面 Markdown 编辑器，基于 Tauri + React + CodeMirror 6。  

## 主要功能

- **三态切换**：编辑 / 编辑+预览 / 纯预览
- **实时预览**：支持 GFM、KaTeX 数学公式、Mermaid 图表
- **代码高亮**：highlight.js 语法高亮，支持数十种编程语言
- **滚动同步**：编辑区和预览区双向滚动联动
- **主题切换**：跟随系统 / 浅色 / 深色

## 开发

```bash
pnpm install
pnpm dev          # 前端 Vite 开发服务器
pnpm tauri dev    # 启动 Tauri 桌面应用
```

## 构建

```bash
pnpm build        # 前端生产构建
pnpm tauri build  # 打包桌面应用安装包
```
